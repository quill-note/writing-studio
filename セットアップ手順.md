# GitHubにアップロードする手順（スマホから）

## 方法A: ブラウザからアップロード（おすすめ）

### 手順1: GitHubのリポジトリを開く
1. Chromeで github.com にログイン
2. `creative-app01` リポジトリを開く

### 手順2: 既存のREADME.mdを削除
1. `README.md` をタップ
2. 右上のゴミ箱アイコン（またはメニュー → Delete）をタップ
3. 「Commit changes」で確定

### 手順3: ファイルをアップロード
1. リポジトリのトップページで「Add file」→「Upload files」をタップ
2. 「choose your files」をタップ
3. ダウンロードした以下のファイルを選択：
   - `index.html`
   - `manifest.json`
   - `sw.js`
   - `README.md`
4. 「Commit changes」をタップ

### 手順4: iconsフォルダをアップロード
1. 「Add file」→「Upload files」をタップ
2. ※ スマホからフォルダごとのアップロードは難しいので、
   アイコンファイルは手順5で別途対応します

### 手順5: iconsフォルダを作成してアイコンをアップ
1. 「Add file」→「Create new file」をタップ
2. ファイル名に `icons/icon-192x192.png` と入力
   → 自動的にiconsフォルダが作られます
3. ただしPNGは直接作成できないので…

### 【簡単な代替方法】
PCがあれば、ZIPファイルを解凍してから一括アップロードが一番簡単です。
PCがなければ、以下の手順でスマホだけで完了できます：

1. ダウンロードした各ファイルを1つずつ「Upload files」でアップ
2. iconsフォルダ内のファイルは以下のURLから直接ダウンロードして
   「Upload files」で `icons/` パスを指定してアップ

## 方法B: PCからアップロード（一番簡単）
1. ZIPファイルをPCに送る（Google Drive等）
2. PCで解凍
3. GitHubの「Upload files」で全ファイルを一括ドラッグ＆ドロップ

## GitHub Pagesを有効にする
1. リポジトリの「Settings」を開く
2. 左メニューの「Pages」をタップ
3. Source → 「Deploy from a branch」
4. Branch → 「main」、フォルダ → 「/ (root)」
5. 「Save」をタップ
6. 数分後に公開されます
